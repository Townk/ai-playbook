# Running playbooks

`run` executes a playbook from the store (by slug) or a file (`--file`), adapting
it to the current project on the way. There are three modes: the default
interactive pager (drive steps free-form), `--assisted` (confirm each step), and
`--auto` (unattended, with rollback). This guide will cover adapt-on-run, the run
modes, and rollback behavior.
